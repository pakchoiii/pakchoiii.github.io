Nand Vth 阈值分布

发表于2025-09-05|更新于2026-06-12|Nand Flash
|总字数:1433|阅读时长:5分钟|浏览量:

SLC

SLC（Single-Level Cell，单层单元）每个 Cell 仅存储 1bit 数据，只需两种状态即可表征：擦除态（Erased） 与 编程态（Programmed）。

从 Vth 分布来看，两种状态各自形成一个独立且清晰的分布峰。读取时只需在两者之间设置一个读取电压 RL1（Vread），通过检测字线（WL）下各 Cell 的实际 Vth 与 RL1 的大小关系，即可区分擦除态与编程态，准确读出 1bit 数据。

TLC

TLC（Triple-Level Cell，三层单元）通过更精细的电压控制，实现单个 Cell 存储 3bit 数据。3bit 共可组合 8 种逻辑状态：111、110、100、000、010、011、001、101。

在 Vth 分布上，这 8 种状态各占一个独立的电压区间，呈现为 8 个明显的分布峰。相比 SLC，TLC 的峰数量大幅增加、峰间间距更窄，对读取电压的精度和区分逻辑的复杂度提出了更高要求 —— 必须通过多组独立读取电压依次检测，才能准确识别每个 Cell 所属的分布峰，从而完整解析出 3bit 数据。

为什么 Vth 呈正态分布？

NAND 闪存的阈值电压（Vth）呈正态分布，其根源在于制造与工作中无法避免的随机性。存储单元内的随机杂质与结构不完美性，导致电子移动行为具有统计随机性，大量单元的 Vth 数据聚合后自然服从正态分布。

具体来说，两点因素起关键作用：

  1. 随机杂质：半导体制造中，沟道和电荷捕获层会残留随机杂质（如金属离子、晶格缺陷），形成局部的”电势陷阱”或”导电阻碍”。不同单元的电子移动阻力因此各不相同，Vth 值随之分散。
  2. 结构不完美：器件结构的微观差异（如氧化层厚度波动、电极界面粗糙）进一步放大随机性 —— 氧化层薄则电子隧穿容易、Vth 偏低；氧化层厚则隧穿困难、Vth 偏高。

由于杂质分布与结构差异均为独立随机事件，而单颗 NAND 芯片通常包含数百万至上亿个单元，根据中心极限定理，大量独立随机变量（各单元的 Vth 值）的统计结果趋近正态分布，呈现”钟形”曲线：多数单元 Vth 集中在均值附近，少数因极端情况落在两端。

阈值电压

可以把 NAND 存储单元想象成一个”电子开关”：**阈值电压（Vth）**就是让这个开关刚好”导通”或”断开”的临界电压值。它是判断存储单元状态、实现数据读写的核心基准。

Vth 的核心作用：定义逻辑状态

阈值电压本质上是区分”0”和”1”的电压界限，随存储类型不同而调整：

  • 电压低于 Vth 时，单元”断开”（对应逻辑 “0”）；电压高于 Vth 时，单元”导通”（对应逻辑 “1”）。
  • SLC 只需 1 个阈值电压即可区分 2 种状态；TLC 需区分 8 种状态，因此设置多组 Vth 区间，但本质仍是靠”电压是否超过阈值”来判断单元的具体逻辑状态。

Vth 对数据读写的关键影响

阈值电压的大小和稳定性，直接决定了 NAND 数据读写的准确性与可靠性：

  • 读取环节：若 Vth 设置不合理（如过于靠近某个分布峰边缘），可能导致逻辑状态误判。
  • 编程/擦除环节：每次写入或擦除的本质，是通过调整单元内电荷量来改变其实际 Vth 值。若阈值电压发生漂移（如长期使用导致电荷泄漏），单元将难以稳定维持目标逻辑状态，进而影响数据存储寿命。
  • 与 Vth 分布的关联：Vth 正态分布是大量单元实际 Vth 的统计结果，读取电压需避开分布峰边缘，才能确保识别准确。

TLC 的 Vth 分布与格雷码

前文提到，TLC 单个 Cell 存储 3bit 时，8 种逻辑状态对应 8 个 Vth 分布峰。每个峰对应的 3 个 bit 按位权重分为三档：

缩写 全称 含义
LSB（LP） Least Significant Bit 最低有效位
CSB（UP） Center Significant Bit 中间有效位
MSB（XP） Most Significant Bit 最高有效位

三者共同定义该峰的逻辑状态。下面以 格雷码 2-3-2 方案为例：

图中最左侧为 Erased（擦除）状态的 Vth 峰。观察相邻峰之间的 bit 变化规律：

  • 相邻峰（P1↔P2、P3↔P4、P5↔P6）之间仅有 1 个 bit 不同，且差异集中在 CSB；
  • 这种”相邻峰仅 1bit 不同”的设计，正是 Gray Code（格雷码） 的核心思想，可有效降低数据读写时的误判概率。

由上表可知，识别 CSB 的 bit 值需要在 RL6、RL4、RL2 分别读取三次，对三次 Reference Read 的结果做异或运算得到最终 bit 值。

具体而言，TLC 三个 bit 与 Page 的对应关系如下：

bit 别名 所需读取次数 对应 RL
LSB LP 2 RL5、RL1
CSB UP 3 RL6、RL4、RL2
MSB XP 2 RL7、RL3

这只是 TLC 众多方案中的一种。常见的 TLC Gray Code 方案包括 2-3-2、1-2-4、1-3-3。不同存储介质根据单元状态数、读写机制和可靠性需求，会采用不同的格雷码方案，其差异直接影响四项核心指标：读写速度、数据可靠性、错误校正复杂度、功耗。
